LeetCode 两数之和问题解析

题目描述

给定一个整数数组 nums 和一个整数目标值 target，请你在该数组中找出 和为目标值 target 的那 两个 整数，并返回它们的数组下标。

你可以假设每种输入只会对应一个答案，并且你不能使用同一个元素两次。

你可以按任意顺序返回答案。

解题思路

方法一：暴力解法

  • 遍历数组中的每一个元素，对于每个元素，再遍历剩下的元素，检查是否存在和为目标值的组合
  • 时间复杂度：O(n²)，空间复杂度：O(1)

方法二：哈希表

  • 使用哈希表存储已经遍历过的元素及其下标
  • 对于当前元素，计算目标值与当前元素的差值，检查哈希表中是否存在该差值
  • 如果存在，返回当前下标和哈希表中存储的下标
  • 时间复杂度：O(n)，空间复杂度：O(n)

AI 解释

算法原理

这个解法使用了哈希表（Map）来优化查找过程。哈希表的特点是可以在 O(1) 的时间复杂度内进行查找操作，这大大提高了算法的效率。

代码执行流程

  1. 初始化哈希表：创建一个空的 Map 对象，用于存储已经遍历过的元素值及其对应的下标。

  2. 遍历数组：从数组的第一个元素开始遍历，对于每个元素：

    • 计算目标值与当前元素的差值 complement
    • 检查哈希表中是否存在这个差值
    • 如果存在，说明我们找到了两个数的和为目标值，返回哈希表中存储的下标和当前元素的下标
    • 如果不存在，将当前元素及其下标存入哈希表中

  3. 返回结果：如果遍历完整个数组都没有找到符合条件的两个数，返回空数组。

为什么这个解法有效

  • 时间复杂度优化：通过哈希表，我们将查找操作的时间复杂度从 O(n) 降低到 O(1)，整个算法的时间复杂度从暴力解法的 O(n²) 优化到 O(n)。
  • 空间换时间：我们使用了额外的哈希表空间来存储已遍历的元素，换取了时间复杂度的降低。
  • 一次遍历：这个解法只需要遍历一次数组，就可以找到答案，相比暴力解法的双重循环，效率更高。

边界情况处理

  • 重复元素：当数组中存在重复元素时，哈希表会存储最后一次出现的元素的下标。但由于我们是从左到右遍历，当遇到第一个符合条件的元素对时就会返回，所以不会影响结果。
  • 空数组：如果数组为空，函数会直接返回空数组。
  • 无解情况：如果遍历完整个数组都没有找到符合条件的两个数，函数会返回空数组。

复杂度分析

  • 时间复杂度：O(n)，其中 n 是数组的长度。我们只需要遍历一次数组，每次遍历中的哈希表操作都是 O(1) 的时间复杂度。
  • 空间复杂度：O(n)，其中 n 是数组的长度。哈希表最多需要存储 n 个元素。
